# Contributing Guide

Thanks for contributing to this demo repository.

## Getting Started

1. Review the [README](README.md) for the current project structure.
2. Create a branch for your work.
3. Keep changes small and easy to review.

## Branching

Use short, descriptive branch names such as `docs/update-readme` or `feature/add-placeholder`.

## Pull Requests

- Summarize what changed and why.
- Mention any follow-up work that is still needed.
- Include test notes, even if the answer is "placeholder only" for this demo.

## Testing

The test setup in this repository is intentionally minimal for demo purposes. Replace the placeholder material in `tests/` with real tests as the project grows.
